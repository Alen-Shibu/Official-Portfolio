What it does

Core Features

Real-time Messaging

WebSocket-powered instant delivery via Socket.io. Messages appear without any page refresh or polling.

Online Presence

Live online/offline indicators for all users. Presence tracked server-side and pushed to all connected clients.

JWT Authentication

Secure stateless auth with HTTP-only cookies. Protected routes on both frontend and backend.

Media Uploads

Image sharing via Cloudinary CDN. Drag-and-drop or file picker, with instant preview before sending.

Persistent History

All conversations stored in MongoDB. Full chat history loads on login, paginated for performance.

Theme Support

Multiple UI themes with user preference stored in localStorage. Clean, responsive design across all devices.

Problem solving

Challenges & Solutions

01

Socket Room Management

Keeping track of which users are online and routing messages only to intended recipients required a server-side Map of userId → socketId, cleaned up on disconnect events.

02

Auth Across HTTP + WebSocket

JWT stored in HTTP-only cookies works seamlessly for REST calls but doesn't auto-attach to WebSocket handshakes. Solved by verifying the token from a handshake query param on the server's connection event.

03

Zustand State Sync

Incoming real-time messages needed to update the active conversation view without refetching. Used Zustand's store to hold the active chat messages and appended new socket events directly to it.
